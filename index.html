Ein Element setzt sich (normalerweise) aus einem öffnenden und einem schliessende Tag zusammen:

Elemente sollten generell immer geschlossen werden, auch die leeren Elemente.

Styles werden an alle auf die gewählten Selektoren "passenden" Elemente (in der Reihenfolge Ihrer Spezifität) zugewiesen.

Selektoren?

..werden benutzt um Elemente in eine Formatierung einzubeziehen. Selektoren können miteinander kombiniert werden.

Spezifität?

..bedeutet, daß unterschiedliche Selektoren unterschiedlich stark gewichtet werden (können).

CSS - Selektoren

Klasse
.hintergrund

..erfasst Elemente die das Attribut class="hintergrund" haben.

Elemente
p

erfasst alle Absätze

ID's
#logo

erfasst das Element, dessen Attribut ID="logo" gesetzt hat.

CSS - Kombination von Selektoren

section.important

Erfasst alle Elemente vom Typ "section" die die Klasse "important" gesetzt haben

#logo.important

Erfasst das Element mit der ID "logo", wenn es die Klasse "important" gesetzt hat.

section#logo.important

Erfasst das Element mir der ID "logo", wenn es vom Typ "section" ist und die Klasse "important" gesetzt hat.

CSS - Kombination von Selektoren

.really.important

Erfasst Elemente die die Klassen "important" und "really" gesetzt haben (AND)

.important, .notice

Erfasst Elemente die die Klassen "important" oder "notice" gesetzt haben (OR).

.dialog .important

Erfasst Elemente die die Klasse "important" gesetzt haben und ein übergeordnetes Element besitzen, welches die Klasse "dialog" hat.

CSS - Weitere Selektoren

:hover

Pseudo-Elemente

CSS - Media queries


@media ("Merkmale und Kriterien") {"angewendete Styles"}

Bei Vorliegen bestimmter Kriterien sollen die angegebenen Styles zur Anwendung kommen.

Ein Beispiel:
@media (min-width: 768px) {p:{color:red}}

..bewirkt, daß sich ab einer Bildschirmgröße von 768px die Farbe der Absätze ändert.

Merkmale können auch mit logischen Operatoren verbunden werden:
@media (min-width: 768px) and (max-width: 1280px) {p:{color:red}}
Ab einer Breite von 1281px soll wieder die "normale" Farbe angewendet werden.

CSS - Zeilenlauf und Positionierung

position: static
..die Standardeinstellung, belässt Elemente im Zeilenlauf

position: absolute
..Elemente wird vom Zeilenlauf losgelöst im nächsthöheren (mit z.B. position:relative definierten) Element erzeugt und kann mit diversen Angaben positioniert und in seiner Größe definiert werden.

position: fixed
..Element wird absolut positioniert, aber alle Angaben zur Ausrichtung beziehen sich auf den Viewport des Browsers. Nützlich für feststehende Elemente in Seiten.

position: relative
..Element soll als Bezugspunkt für weitere (meist absolute) Positionierungen dienen, es kann aber auch selbst im Zeilenlauf positioniert werden.

CSS - Zeilenfluss der Elemente

float: none
..die Standardeinstellung, Element wird nicht verschoben

float: left / right
..Element wird an die linke oder rechte Kante des (nächst höheren) Elternelements verschoben.

float: inherit
..Element erbt die float-Eigenschaft des Elternelements

clear: none / left / right / both / inherit
..wird verwendet um zu verhindern, daß ein Element von floatenden anderen Elementen verdeckt werden kann.

Anmerkung: floats..
dienen nicht dazu, den Textfluss z.B. in einem Absatz zu steuern. Dafür verwendet man text-align

Challenge - STEP16

"NOT WORK" - WTF is CORS?
(..ist ein browserbasierter Sicherheitsmechnismus)

Irgendwie kommen da keine Daten an.
In den Developer-Tools in der Console sehen wir: 

 XMLHttpRequest cannot load https://www.abgeordnetenwatch.de/api/parliament/berlin/candidates.json. 
No 'Access-Control-Allow-Origin' header is present on the requested resource. 
Origin 'http://localhost:3000' is therefore not allowed access.

Für eine Einführung in CORS fehlt jetzt die Zeit,
aber hier gibt es detaillierte Infos.

Zusammenfassend: Der Server, der "cross-site" angefragt wird, liefert eine Information zurück, welche Domains ihn fragen dürfen und der Browser entscheidet anhand dessen und anhand der Herkunfts-Domäne, ob dieser Request erlaubt ist.
In unserem Fall erlaubt dies Abgeordnetenwatch leider (noch) nicht, aber vielleicht macht man sich auch Sorgen um Überlastung, da Requests dort manchmal 30 Sekunden dauern können...

Schade - aber wir können ja die Datei auch einfach mal im Browser von der Url direkt runterladen und dann
lokal ablegen unter "app/candidates.json"

Dann müssen wir aber den Request in app.component.js noch ändern auf: 

http.get('app/candidates.json')
(Nun werden die json-Daten direkt lokal von unserem Testserver ausgeliefert.)
